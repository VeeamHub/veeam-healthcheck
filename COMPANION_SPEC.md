# VHC Monitor — Companion Spec

## Overview

`vhc-monitor` is a standalone Python-compiled Windows executable that performs continuous health monitoring of Veeam Backup & Replication servers. It is bundled alongside `VeeamHealthCheck.exe` and managed through the VHC GUI and CLI.

## Bundle Layout

```
<VHC install dir>/
  VeeamHealthCheck.exe          # Main health check application
  vhc-monitor.exe               # Companion monitor (optional; bundled at release time)
  BUNDLE_MANIFEST.json           # Version + SHA256 of bundled vhc-monitor
```

VHC detects the monitor by checking for `vhc-monitor.exe` in the same directory as `VeeamHealthCheck.exe` (`Environment.ProcessPath` directory).

## Install Directory

```
%AppData%\VeeamHealthCheck\monitor\
  vhc-monitor.exe               # Copied from bundle during setup
  vhc-monitor.yaml              # Generated config
  veeam-monitor-state.json      # Written by vhc-monitor after each run
```

- **No admin required** — `%AppData%` is user-writable.
- Matches the existing `%AppData%\VeeamHealthCheck\creds.json` credential storage pattern.

## Config File: `vhc-monitor.yaml`

Generated by `CVhcMonitorIntegration.GenerateConfig()`. Example:

```yaml
# vhc-monitor configuration — generated by VeeamHealthCheck
global:
  timeout_seconds: 30
  retry_count: 2

servers:
  - name: vbr-primary
    type: vbr
    url: "https://vbr-server:9419"
    username: "DOMAIN\\admin"
    password: "secret"
    api_version: "1.3-rev1"
    verify_ssl: false

output:
  json_stdout: true
  webhook:
    type: "ntfy"
    url: "https://ntfy.sh/your-topic"
    min_severity: "warning"

monitors:
  repo_health:
    enabled: true
  retention:
    enabled: true
  worker_health:
    enabled: true
```

### Server entry fields

| Field        | Required | Description                                      |
|-------------|----------|--------------------------------------------------|
| name        | yes      | Friendly label                                   |
| type        | yes      | `vbr` or `vbaws`                                 |
| url         | yes      | REST API endpoint, e.g. `https://host:9419`      |
| username    | yes      | Login username                                   |
| password    | yes      | Login password (plaintext in YAML)               |
| api_version | no       | Defaults to `1.3-rev1`                           |
| verify_ssl  | no       | Defaults to `false`                              |

### Webhook types

`ntfy`, `teams`, `slack`, `pagerduty`

## CLI Commands VHC Invokes

| Purpose            | Command                                                            |
|--------------------|--------------------------------------------------------------------|
| Full check run     | `vhc-monitor.exe all --config "<ConfigPath>"`                      |
| Test connection    | `vhc-monitor.exe test-connection --config "<ConfigPath>"`          |
| Get version        | `vhc-monitor.exe --version`                                       |

## Scheduled Task

- **Task Name**: `VHC Monitor`
- **Interval**: Every 5 minutes
- **Execution Limit**: 4 minutes
- **RunLevel**: `Limited` (current user, no elevation)
- **Registration**: `Register-ScheduledTask` via PowerShell

## State File: `veeam-monitor-state.json`

Written by `vhc-monitor.exe` after each run. Read by VHC to display last-run status.

```json
{
  "last_run": "2026-04-01T20:00:00Z",
  "summary": "All checks passed",
  "finding_count": 0
}
```

| Field         | Type     | Description                           |
|--------------|----------|---------------------------------------|
| last_run     | ISO 8601 | Timestamp of last completed run       |
| summary      | string   | Human-readable result summary         |
| finding_count| int      | Number of findings/alerts             |

## Status Detection Logic

VHC determines monitor state through this cascade:

1. **Not bundled** — `vhc-monitor.exe` not found next to `VeeamHealthCheck.exe`
2. **Available — not set up** — bundled exe exists but not installed to `%AppData%` or task not registered
3. **Running** — installed exe exists AND scheduled task `VHC Monitor` is registered

## VHC CLI Integration

```
VeeamHealthCheck.exe /monitor:setup     # Install + register task
VeeamHealthCheck.exe /monitor:run       # Trigger immediate check
VeeamHealthCheck.exe /monitor:status    # Print status summary
VeeamHealthCheck.exe /monitor:disable   # Remove scheduled task
```
